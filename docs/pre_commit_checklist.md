# Pre-commit Checklist\n\nAntes de enviar cambios, verifica lo siguiente:\n\n1. \[ \] Ejecuta linters/tests relevantes:\n   - Backend: cd services/api && ./.venv/Scripts/python manage.py test (o pytest si aplica)\n   - Web: cd apps/web && npm run lint\n   - Operator App: cd apps/operator-mobile && npm run lint\n   - Android: cd apps/android && ./gradlew lintDebug assembleDebug\n2. \[ \] Actualiza migraciones si cambiaste modelos (python manage.py makemigrations).\n3. \[ \] Revisa git status y elimina archivos temporales/secretos.\n4. \[ \] Formatea/ordena código (Prettier, Black, ktlint, etc.).\n5. \[ \] Sincroniza env docs si agregaste variables (docs/setup_v2.md, .env.example).\n6. \[ \] Añade notas en docs/CHANGELOG o progreso si aplica.\n7. \[ \] Prueba funciones críticas manualmente (autenticación, uploads, pagos, mapa).\n8. \[ \] Ejecuta 
pm run build (web) y ./gradlew assembleDebug (android) si hubo cambios de build.\n9. \[ \] Actualiza README/diagramas cuando cambie arquitectura.
